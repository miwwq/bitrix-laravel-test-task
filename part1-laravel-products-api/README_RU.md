# Часть 1. Laravel: `GET /api/products`

## Что делает эндпоинт

Пример запроса:

```http
GET /api/products?category_id=1&price_from=100&price_to=500&in_stock=1&page=1&per_page=20
```

Эндпоинт возвращает товары постранично, фильтрует по категории, цене и наличию, а также сразу отдает связанные данные категории и остатка.

## Почему Eloquent

Для тестового задания я выбрал Eloquent, потому что он лучше показывает структуру доменной модели: `Product`, `Category`, `ProductStock` и связи между ними. При этом запрос остается экономным:

- выбираются только нужные колонки через `select()`;
- `per_page` ограничен максимумом 100;
- используется `simplePaginate()`, чтобы не делать дорогой `COUNT(*)` по большому каталогу;
- связанные данные загружаются через `with()`, поэтому нет N+1.

Если в реальном проекте понадобится очень глубокая навигация по каталогу, вместо `page` можно перейти на `cursorPaginate()` или seek pagination по `id`. Это лучше для страниц вроде 50000-й, потому что БД не нужно пропускать большой offset.

## Индексы

Основные индексы:

- `products.sku` уникальный, чтобы быстро искать товар по артикулу;
- `products(category_id, price, id)` для фильтра категории и диапазона цены;
- `products(price, id)` для фильтра только по цене;
- `product_stocks(quantity, product_id)` для фильтра наличия;
- `product_stocks.product_id` является primary key, потому что у товара одна запись остатка.

Идея простая: поля, по которым фильтруем миллион товаров, должны быть проиндексированы. `id` добавлен в составные индексы, потому что выдача стабильно сортируется по `id`.

## N+1

Проблема N+1 была бы, если бы мы получили 20 товаров, а потом для каждого отдельно сделали запрос категории и остатка. Здесь используется:

```php
->with([
    'category:id,name,slug',
    'stock:product_id,quantity',
])
```

Laravel сделает один запрос за товарами, один за категориями для этих товаров и один за остатками. То есть количество запросов не растет вместе с количеством товаров на странице.

## Валидация и ошибки

Валидация вынесена в `ProductIndexRequest`. Если параметры некорректны, API возвращает JSON с HTTP-кодом `422`:

```json
{
  "message": "Invalid request parameters.",
  "errors": {
    "per_page": ["The per page field must not be greater than 100."]
  }
}
```
